# 直接效应、间接效应、总效应入门

## 一句话理解

一个变量影响结果，可能有两条路：直接影响，或者先影响中介变量，再通过中介影响结果。前者叫直接效应，后者叫间接效应，两者加起来叫总效应。

## 本项目在哪里使用

- C22 在 calculate_effects 中明确计算长度、格式、能力差等变量的直接效应、间接效应和总效应。
- C22 还用 bootstrap 给这些效应提供区间估计。

## 为什么本项目必须区分这三种效应

以 ability_diff 为例，本项目不是只关心：

“能力更强的模型是不是更容易赢？”

还关心：

- 它是直接因为能力更强而赢，还是
- 因为能力更强的模型往往写得更长、更会用标题和粗体，于是通过这些中介间接获胜？

只有把直接效应和间接效应拆开，才能真正谈“机制”。

## 间接效应怎么计算

最基础的链式中介里，间接效应通常是“前半段路径系数 × 后半段路径系数”。

例如：

- ability_diff -> token_diff_ab 的系数
- token_diff_ab -> winner_a 的系数

两者相乘，就是“能力差通过长度差影响偏好”的间接效应。

## 本项目中的典型效应

C22 重点关注：

- 长度直接效应
- 标题密度直接效应
- 粗体密度直接效应
- ability_diff 通过长度和格式中介产生的间接效应
- format_tendency_diff 通过标题密度、粗体密度产生的间接效应

这正好对应本研究的核心问题：偏好不仅由“内容更长”决定，也可能由“更会格式化”决定。

## 总效应怎么理解

总效应 = 直接效应 + 所有间接效应

如果一个变量的直接效应不大，但总效应明显，说明它更像一个“上游驱动因素”，主要通过中介传导影响结果。

## 为什么间接效应常配 Bootstrap

间接效应是两个系数的乘积，它的分布往往不够对称，直接用正态近似可能不稳定。所以本项目用 bootstrap 给它提供 95% CI，这是一种很常见也很合理的做法。

## 零基础常见误区

- 间接效应不等于“更弱的效应”，它只是“通过中介发生”的效应。
- 直接效应为零，不代表变量没用，可能只是全部作用都走中介路径。
- 总效应小，也可能是正向和负向路径互相抵消的结果。

## 在本研究中的作用

直接/间接/总效应的拆分，是项目从“发现偏好”走向“解释偏好机制”的关键一步。它让研究者能够更具体地说：长度偏好是直接发生的，还是被模型能力和格式策略放大的。
