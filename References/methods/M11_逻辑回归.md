# 逻辑回归入门

## 一句话理解

逻辑回归用于结果变量只有两类的情形，例如“获胜/未获胜”。它通过建模概率的对数赔率，来分析多个解释变量和二值结果之间的关系。

## 本项目在哪里使用

- C18 用嵌套逻辑回归分析长度净效应和格式净效应。
- C19 用调整逻辑回归检验长度效应稳健性。
- C20 用调整逻辑回归检验格式效应稳健性。

## 为什么这里不能直接做线性回归

本项目的核心结果变量 winner_a 只有两种状态：

- A 获胜记为 1
- A 未获胜记为 0

线性回归可能会预测出小于 0 或大于 1 的概率，而逻辑回归天生适合处理二值结果。

## 逻辑回归在做什么

它不是直接拟合概率 p，而是拟合：

logit(p) = log(p / (1 - p))

这样做的好处是：

- 左边可以在整个实数范围变化。
- 右边仍然能通过逆变换回到 0 到 1 的概率。

## 在本项目里怎么用

项目里的逻辑回归通常把 winner_a 当作结果变量，把下面这些量当作解释变量：

- 长度差
- 标题/列表/粗体密度差
- 任务类型和评分维度
- 模型能力差、冗长度倾向差、格式倾向差

这样做的目标是：在控制混淆因素后，判断长度或格式本身是否仍然与偏好有关。

## 嵌套模型是什么意思

C18 不是一次性把所有变量全丢进去，而是做 M0 到 M3、F0 到 F3 的嵌套模型。

它的思想是：

1. 先看粗效应。
2. 再逐层加入任务控制、评分维度控制、模型层控制。
3. 观察核心系数如何衰减。

这样可以更清楚地展示“混淆控制之后，效应还剩多少”。

## 本项目里还记录了什么

除了系数本身，逻辑回归部分还会报告：

- OR
- AME
- Wald 部分相关 r
- McFadden pseudo R²
- AIC 和 BIC

这说明项目不是只关心显著性，而是同时关心效应大小、解释力度和模型拟合。

## 在本研究中的作用

逻辑回归是项目进入“净效应”分析的核心工具。它把前面描述统计和检验得到的现象，推进到“在控制一系列混淆因素后，这种偏好是否仍然存在”的层面。
