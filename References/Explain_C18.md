# C18_pure_effect 代码详解

> 来源文件：`Codes/C18_pure_effect.py`  
> 产出文件：`Reports/R16_pure_effect_report.txt`、`Tables/T05_pure_length_net_effect_summary.csv`、`Tables/T06_pure_format_net_effect_summary.csv`、`Pictures/P08_length_confounding_attenuation.png`、`Pictures/P09_format_net_effect_heatmaps.png`

---

## 一、研究问题与分析目标

C16 和 C17 通过 Wilcoxon 检验证明了"胜者更长"和"胜者格式更多"的现象在统计上是显著的，但这些"粗效应"混入了若干混淆来源：

1. **模型能力混淆**：能力更强的模型更容易赢，而能力更强的模型往往也倾向于生成更长、更有格式的回答（Spearman r(胜率, 均长) = 0.72，r(胜率, 均格式) = 0.53，均高度显著）。
2. **问题特征混淆**：复杂任务的提示词本身更长，可能驱动模型生成更长的回答。
3. **长度-格式共线性**：更长的回答天然含有更多格式元素，格式偏好可能只是长度偏好的投影。

C18 的目标是在控制上述三类混淆后，分别估计长度和格式偏好的**净效应（Net Effect）**，并量化"到底有多少粗效应是真实偏好、多少是被混淆放大的假象"。

## 二、方法选择

### 2.1 为什么用逻辑回归而非继续用 Wilcoxon

Wilcoxon 是无参数的配对检验，适合检验"差值中位数是否为零"，但无法同时控制多个协变量。要分离混淆变量的影响，必须把多个因素同时纳入一个回归模型。因此 C18 转向 **多元逻辑回归（Logistic Regression）**：

- **因变量**：`winner_a`（model_a 获胜 = 1，model_b 获胜 = 0）——这是一个二元结果变量，逻辑回归的标准场景。
- **自变量**：配对差值特征（长度差、格式密度差）+ 混淆控制变量组。
- 核心关注点是 $\beta$ 系数和对应的 **优势比（OR）**，它们可以解释为"在控制其他变量后，某特征每增加一个标准差，胜出概率的相对变化"。

### 2.2 嵌套模型设计（逐步加入混淆）

单层逻辑回归无法揭示"混淆来自哪里"。C18 采用**嵌套层次递进**的设计：

| 层次 | 长度系列 | 格式系列 | 新增控制变量 |
|------|----------|----------|------------|
| 基准层 | M0 | F0 | 仅核心效应变量（长度差 / 格式密度差；F 系列同时控制长度差） |
| 第一层 | M1 | F1 | + 任务类型布尔（creative_writing / IF / math / code）+ 问题长度（user_tokens）+ 轮次数（turns） |
| 第二层 | M2 | F2 | + 7 维 criteria 布尔值（complexity、creativity 等问题复杂度标签） |
| 最终层 | M3 | F3 | + 模型能力差（ability_diff）+ 词冗性差（verbosity_diff）+ 格式风格差（format_tendency_diff） |

每加入一层，关注核心效应变量的 $\beta$ 系数如何变化。如果 M3 比 M0 的 OR 大幅缩水，就说明粗效应中有相当比例是混淆贡献的。

**混淆比例（Confounding Share）**公式：

$$\text{混淆比例} = \frac{OR_{M0} - OR_{M3}}{OR_{M0} - 1}$$

- $< 30\%$：效应主要为真实偏好  
- $30\%\sim70\%$：部分受混淆放大  
- $> 70\%$：效应主要由混淆驱动

### 2.3 格式分析中固定控制长度

格式密度已定义为 $\text{count}_f / \text{tokens}$（相对量），但在逻辑回归模型中，**z_token_diff_ab 在 F0～F3 的所有层次中都作为固定协变量出现**。原因是：格式密度和长度差并非完全正交，固定长度后格式密度的 OR 才真正代表"排除长度影响后，结构化表达习惯本身的贡献"。

### 2.4 z-score 标准化

所有连续型变量在进入逻辑回归前都通过 `zscore_series()` 做了 z-score 标准化。这样：
- 不同量纲的变量系数可以直接比较大小（均代表"每增加一个标准差"的效应）。
- 数值尺度一致，缓解逻辑回归对极端值敏感的问题。
- 常数列（标准差为零）自动设为 0，不报错。

### 2.5 效应量指标

| 指标 | 含义 |
|------|------|
| $\beta$ | 标准化对数优势比系数 |
| OR | 优势比，OR > 1 表示正向效应，OR = 1 表示无效应 |
| AME（平均边际效应）| $\approx \beta \times \bar{p}(1-\bar{p})$，近似每单位 SD 对胜率的绝对增量，更直观 |
| Wald r | $= z_{Wald}/\sqrt{n}$，标准化效应量，类比相关系数，可跨子集比较 |
| McFadden $R^2$ | 全模型拟合优度，衡量模型整体解释比例 |

## 三、实现逻辑

### 3.1 整体流程

```
全量 optimized_data.parquet
        ↓
build_model_stats()          ← 计算每个模型的全局胜率/均长/均格式（不在子集内重估）
        ↓
validate_model_confounding() ← 前置验证：胜率与词冗性/格式的 Spearman r
        ↓
for 每个子集（全量 + 16 个纯净分区）：
    load_subset()            ← 读取 C13 子集，展平列，添加配对特征
        ↓
    run_nested_models_length()   ← M0→M3 逐步逻辑回归（长度系列）
    run_nested_models_format()   ← F0→F3 逐步逻辑回归（格式系列）
        ↓
汇总 → 报告 / CSV 表 / 衰减图 / 热图
```

### 3.2 模块与关键函数

#### `load_data_global()` 与 `build_model_stats()`

全量数据只读一次，用于计算每个模型的三个全局代理变量：

- `win_rate`：全局胜率（能力代理）
- `mean_tokens`：平均回复长度（词冗性代理）
- `mean_format`：平均总格式元素数 = header + list + bold 之和（格式风格代理）

这三个统计量**必须从全量数据计算，不得在子集内重估**，原因是小子集中模型出现次数有限，重估的胜率方差极大，会引入噪声而非控制混淆。

#### `add_pair_features()`

在每个子集 DataFrame 上构造所有配对差异特征列：

```python
# 长度差（绝对量）
df["token_diff_ab"] = df["a_tokens"] - df["b_tokens"]

# 格式密度差（归一化量，分母 clip(lower=1) 防止除零）
df["header_density_diff"] = (a_header_count / a_tokens.clip(1)) - (b_header_count / b_tokens.clip(1))

# 模型能力/风格差（从全局 model_stats 映射到每一行）
df["ability_diff"]         = df["wr_a"]        - df["wr_b"]
df["verbosity_diff"]       = df["verbosity_a"] - df["verbosity_b"]
df["format_tendency_diff"] = df["fmt_a"]       - df["fmt_b"]

# 结果变量
df["winner_a"] = (df["winner"] == "model_a").astype(int)
```

注意：格式密度差定义与 C17 中的 `oriented_winner_density_difference` 略有不同——C18 用的是 $a - b$（不取 winner 方向），因为逻辑回归中 winner_a 已经显式作为因变量，不需要"先定向再求差"。

#### `_standardize()` 与 `_active()`

`_standardize(df)` 对 `_CONT_VARS` 列表中的所有连续型变量添加 `_z_前缀` 列。`_active(s, cols)` 过滤掉值全相同的常数列，防止它们进入模型引起奇异矩阵报错。

#### `run_nested_models_length()`

四层嵌套拟合的核心实现：

```python
X0 = s[[z_tgt]]                                          # M0: 仅长度差
X1 = s[[z_tgt] + task_bool + prompt_z]                  # M1: +问题特征
X2 = pd.concat([X1, s[crit_cols]], axis=1)              # M2: +criteria
X3 = pd.concat([X2, s[model_z]],   axis=1)              # M3: +模型层

fits = {lbl: _fit_logit(X, y) for lbl, X in [...]}      # 四层均拟合
```

对每层提取 `z_token_diff_ab` 的系数、SE、p 值，再计算 OR、AME、Wald r，最后用 M0 和 M3 的 OR 计算混淆比例。

#### `run_nested_models_format()`

结构与长度系列完全平行，区别有两点：

1. **基准变量是格式密度差三元组**（header/list/bold），加上固定协变量 `z_token_diff_ab`；
2. **最低样本量门槛更高**（100 对，而长度系列是 30 对）——多变量模型奇异矩阵风险更大。

同一次拟合中三个密度变量同时入模型，每个系数代表"控制其他格式类型和长度后的净效应"。

#### `_fit_logit()`

封装 `statsmodels.Logit`，自动：
- 将所有输入列强制转为 `float64`（防止 bool/object 列报错）
- 调用 `sm.add_constant()` 添加截距
- 最多迭代 300 次，忽略收敛警告（小子集常见，不中断主流程）
- 收敛失败返回 `None`，上层函数安全跳过

McFadden $R^2 = 1 - \text{llf}/\text{llnull}$，也在此函数中计算。

#### `validate_model_confounding()`

在开始嵌套模型之前，先用 Spearman 相关系数验证"控制 ability_diff 是否必要"。如果 |r| > 0.3，则认为共线性成立，控制步骤有依据。

#### 可视化

| 函数 | 图形类型 | 编码方式 |
|------|---------|---------|
| `plot_length_attenuation()` | 哑铃图（Dumbbell Chart） | 橙点 = 粗效应 OR，绿点 = 净效应 OR，水平线显示衰减幅度，右侧文字标注混淆比例 |
| `plot_format_heatmaps()` | 双热图 | 左：净 OR（F3），颜色以 1.0 为中心双向发散；右：混淆比例（0%～max%），颜色深度表示混淆程度 |

### 3.3 产出文件

| 文件 | 内容 |
|------|------|
| `Reports/R16_pure_effect_report.txt` | 完整嵌套模型逐子集报告，含前置验证、全部系数表、混淆比例小结 |
| `Tables/T05_pure_length_net_effect_summary.csv` | 长度净效应汇总宽表（每子集一行） |
| `Tables/T06_pure_format_net_effect_summary.csv` | 格式净效应汇总长表（每子集×特征一行） |
| `Pictures/P08_length_confounding_attenuation.png` | 长度粗→净 OR 衰减哑铃图 |
| `Pictures/P09_format_net_effect_heatmaps.png` | 格式净 OR 与混淆比例双热图 |

## 四、结论分析

### 4.1 前置验证：混淆控制的必要性已证实

报告首先对 53 个模型做了能力-风格相关性验证：

- Spearman r（胜率 ~ 均长）= **0.72**（p < 0.001）
- Spearman r（胜率 ~ 均格式）= **0.53**（p < 0.001）

这说明"能力强的模型确实更长更有格式"，不控制就会把能力优势误归结为长度/格式偏好。能力最强的 gemini-2.5-pro 胜率 74.2%，均长 1950 tokens，均格式元素 61.3 个，是这种共线性的典型体现。

### 4.2 长度净效应：真实偏好确实存在，但幅度被高估了

**全量数据**结果：

| 指标 | M0（粗效应） | M3（净效应） |
|------|------------|------------|
| OR | 1.9045 | 1.3551 |
| Wald r | 0.1885 | 0.0817 |
| AME | 0.161 | 0.076 |
| 混淆比例 | — | **60.7%** |

解读：控制问题特征、任务难度和模型能力风格后，"胜者更长"仍然显著（$p < 10^{-100}$），但 OR 从 1.90 降到 1.36，有约 61% 的粗效应来自混淆——主要是因为更好的模型本身就更长，而不是因为"更长"本身更受喜爱。净效应每增加一个 SD 的长度差，胜率提升约 7.6 个百分点。

**各子集横向比较**（按净 OR 降序）：

| 子集 | 净 OR | 混淆比例 | 解读 |
|------|-------|---------|------|
| 创意+指令 | **1.83** | 27.5% → 真实偏好 | 混合任务对长度偏好最真实，混淆最少 |
| 创意+代码 | 1.47 | ~0%（几乎无混淆）| 长度偏好几乎全是真实的 |
| 仅代码 | 1.37 | 47.9% → 部分混淆 | 代码任务中真实长度偏好仍中等 |
| 全量 | 1.36 | 60.7% → 部分混淆 | 代表均值，混淆中等 |
| 仅数学 | 1.18 | 71.2% → 主要混淆 | 数学任务中长度偏好主要是能力混淆 |
| 指令+数学 | 1.01（不显著）| 96.5% → 主要混淆 | 几乎全是混淆，净效应消失 |

**关键发现**：数学类任务的长度偏好混淆程度最高——因为会数学、解题长的模型本身能力更强，"更长的数学回答赢"在控制了模型能力后基本消失；而创意写作类任务对长度的真实偏好更强，可能因为读者确实喜欢更丰富的创意内容。

M0→M3 的层间变化揭示混淆主要来自 **M2→M3** 这一跳（加入模型层变量），而非来自问题特征或 criteria，这与前置验证的结论吻合。

### 4.3 格式净效应：三类格式的命运各不相同

**全量数据**格式净效应：

| 格式特征 | 粗 OR（F0） | 净 OR（F3） | 混淆比例 | 机制 |
|---------|-----------|-----------|---------|------|
| 标题密度 | 1.1133 | **1.0606** | 46.5% 部分混淆 | 真实偏好保留约一半 |
| 列表密度 | 0.9398（负向！）| 1.0249 | **141.3% 主要混淆** | 粗效应是负的，控制后翻正——混淆反向掩盖了真实偏好 |
| 粗体密度 | 1.1570 | **1.0855** | 45.6% 部分混淆 | 真实偏好保留约一半，最稳健 |

**列表密度的反常现象值得特别注意**：F0 中列表的粗 OR 是 0.94（负效应），但控制模型能力后翻转为正 1.02。这意味着：在原始数据里"列表更多的回答反而输"，但这是因为某些使用大量列表的低能力模型经常输，掩盖了"同等能力下列表增加会正向影响胜率"这一现象。这是混淆方向反转（辛普森悖论的一种表现），只有逻辑回归能揭示，Wilcoxon 无法发现。

**分任务对比**：

| 场景 | 粗体密度净效应 | 机制判断 |
|------|-------------|---------|
| 仅代码 | OR=1.09，混淆仅 13%（真实偏好） | 代码中粗体强调是独立的表达优势 |
| 数学+代码 | OR=1.15，混淆仅 2%（真实偏好） | 同上 |
| 仅创意写作 | OR≈1.00，混淆 93%（主要混淆）| 创意文本中粗体格式效应几乎全是能力混淆 |
| 仅数学 | OR=1.03（不显著），混淆 85% | 数学中粗体效应基本被能力吸收 |

**标题密度**：在任务差异上，创意写作和创意+指令类子集中标题净效应极小甚至为负，而代码类和非类别化场景中正效应更明显——意味着"标题带来有条理感"在技术类任务更受认可。

**综合结论**：格式偏好中最稳健的净效应来自**粗体**（尤其在代码/技术类任务中），其次是**标题**；**列表**的净效应在控制混淆后最弱，且存在方向逆转的陷阱，与 C17 中列表密度检验经常不显著的结论相呼应。

### 4.4 模型拟合质量

所有子集的 M0/F0 的 McFadden $R^2$ 较低（0.02～0.05），M3/F3 提升至 0.07～0.15，说明：

1. 单靠长度或格式特征，对人类偏好的解释力有限（符合预期，偏好是多因素的）。
2. 加入模型层变量后解释力显著提升，进一步证明模型能力是最重要的混淆变量。
3. 即使 M3/F3 只能解释 8%～15% 的方差，核心效应变量的 OR 仍然显著，说明长度/粗体偏好是真实存在的独立信号，而不是噪声。

### 4.5 三份报告（C16/C17/C18）的证据链

| 分析 | 方法 | 发现 |
|------|------|------|
| C16 | Wilcoxon 符号秩 | 长度偏好普遍显著，r_rb ≈ 0.33～0.41（中效应） |
| C17 | Wilcoxon + 密度辅助 + 卡方 | 三类格式均显著，但列表密度效应弱且不稳定 |
| C18 | 嵌套逻辑回归 | **净效应确认**：长度真实偏好 OR 约 1.35（全量）；粗体最稳健；列表部分是混淆假象 |

这三个层次形成递进的证据链：现象存在 → 效应量化 → 净效应与混淆分解。C18 的核心贡献是把"多少效应是真实偏好"从"总效应有多大"中分离出来，为后续因果建模（C22 SEM）提供了干净的估计值。

## 五、设计细节与注意事项

1. **全局统计量不在子集内重估**：这是防止小样本偏差的关键设计。如果在"仅数学"子集中只用 3000 条数据估计胜率，方差极大，结果不可靠。
2. **MIN_PAIRS 差异**：长度模型 30，格式模型 100——格式模型有 4 个以上连续协变量，奇异矩阵风险更高，需要更多样本保证数值稳定性。
3. **混淆比例超过 100% 或为负**的情况（例如"列表密度混淆比例=141%"，"创意+数学列表密度=-147%"）是有意义的：超过 100% 说明混淆不仅放大了效应，还把方向搞错了（辛普森悖论）；负值说明控制后效应反而增强，原始数据中存在反向掩盖。
4. **逻辑回归中的 `has_constant="add"`**：强制在 X 已有常数列时仍添加截距，避免某些含全 1 列的数据框报错。
5. **z-score 中常数列设为 0**（`stats_utils.zscore_series`）：确保稀有子集中某一类别全相同时不报错，但该列实际不参与回归（被 `_active()` 过滤掉）。
