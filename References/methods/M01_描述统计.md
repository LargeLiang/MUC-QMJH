# 描述统计入门

## 一句话理解

描述统计不是为了证明因果，也不是为了做显著性检验。它的任务是先把数据本身的样子说明白，比如平均值、标准差、范围、胜率、样本占比各是多少。

## 本项目在哪里使用

- C14 用它描述不同长度差分箱中的胜率、样本数、样本占比。
- C15 用它描述不同格式特征和格式密度下的胜率变化。
- C22 用它汇总外生层、中介层、结果层变量的均值、标准差、分位数和缺失率。

## 它在回答什么问题

当我们拿到一份数据时，最先要回答的通常不是“显著不显著”，而是：

- 数据多不多。
- 分布偏不偏。
- 哪些值常见，哪些值极端。
- 某种现象大概出现了多少次。
- 两个组里谁的平均表现更高。

如果这些最基础的问题都没有先说清楚，后面的检验和建模就容易失去上下文。

## 本项目里最常见的描述统计量

- 均值：一组数的平均水平。
- 标准差：一组数离均值有多分散。
- 最小值和最大值：分布边界。
- 中位数和四分位数：更稳健地看分布中心和分布宽度。
- 胜率：某个条件下 is_winner 的平均值，本质上就是“赢的比例”。
- 样本占比：某个分组里的样本数除以总样本数。
- 缺失率：某个变量有多少比例没有观测值。

## 如何计算

如果把某组样本记作 x1, x2, ..., xn，那么：

- 均值 = 所有数相加后再除以 n。
- 样本标准差 = 看每个数和均值差多远，再做平均并开方。
- 胜率 = 获胜记为 1、未获胜记为 0 后的平均值。
- 样本占比 = 该组样本数 / 总样本数。

所以你会发现，很多“比例”其实都可以转成 0 和 1 的均值来理解。

## 在本项目里怎么读结果

举例说，C14 里的某个长度差分箱如果：

- win_rate = 0.62，说明这个分箱里较长回答获胜的比例约为 62%。
- sample_proportion = 0.08，说明这个分箱只占总样本的 8%。

这时不能只看胜率高不高，还要看它是不是建立在足够样本上。一个样本占比很低的高胜率分箱，解释时要更谨慎。

## 为什么描述统计很重要

- 它帮助你发现异常值和偏态分布。
- 它让你知道后续检验适不适合做。
- 它能提醒你“看起来很强的结论”是不是只来自很少量样本。
- 它为后面的因果和机制分析提供背景。

## 在本研究中的作用

本项目研究的是“人类是否偏好更长、或更有格式的 LLM 输出”。在真正进入检验之前，必须先用描述统计回答：长度差分布如何、格式元素多不多、不同区间样本是否充足。描述统计就是整条分析链的第一层地基。
