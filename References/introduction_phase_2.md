# 第二阶段：描述性分析——偏好的规律长什么样？

> **受众**：无统计学背景的本科生  
> **目标**：理解如何用可视化和基础统计量描述偏好规律，了解"相关性"与"因果性"的区别

---

## 1. 描述性分析的目的

经过第一阶段的数据清洗，我们有了 108,171 条整洁的配对记录。现在的问题是：**这堆数据里藏着什么规律？**

描述性分析（Descriptive Analysis）就是用统计图表和汇总指标，把数据的大致形态展示出来，回答：

- 人类有多经常选择更长的那个回答？
- 有格式的回答（标题、列表、粗体）比纯文字回答被选中的概率高多少？
- 这个规律在数学题、创意写作、代码任务中是否一样？

> **注意**：描述性分析只告诉我们"有没有规律"，**不能**告诉我们"为什么有这个规律"。

---

## 2. 数据解缠化：为什么样本量翻倍？

报告里你会看到"样本量：157,940（解缠化后）"，但原始配对数只有 78,970。为什么？

### 2.1 什么是"解缠化"（Detangling）

每条原始记录是一个 **A vs B 的配对**：

```
一条原始记录：
  model_a = GPT-4o,  回复长度 = 800 tokens
  model_b = Claude,  回复长度 = 200 tokens
  winner  = model_a
```

为了分析"获胜回复的特征是什么"，我们把每条记录**拆成两条**：

```
拆分后记录 1：
  该模型 = GPT-4o,  长度 = 800,  结果 = 胜

拆分后记录 2：
  该模型 = Claude,  长度 = 200,  结果 = 负
```

这样每条记录就代表"一个模型的一次回答"，便于比较"获胜方"和"落败方"的特征分布。

> **比喻**：就像把"甲队 vs 乙队，甲赢"这条足球比赛记录，拆成"甲队（胜）"和"乙队（负）"两条球队记录，方便统计各队的得分率。

---

## 3. 长度偏好分析（C14）

### 3.1 核心指标：长度差值

分析方法：把每对配对转化为一个**差值**：

$$\delta_i = \text{tokens}(\text{获胜模型}) - \text{tokens}(\text{落败模型})$$

- 如果 $\delta_i > 0$：这一对中"更长的那个"赢了
- 如果 $\delta_i < 0$：更短的那个赢了
- 如果 $\delta_i = 0$：两者一样长

然后把所有配对按 $\delta_i$ 的大小分成若干组（**分箱**），计算每组里"更长者获胜"的比例（**胜率**）。

### 3.2 分箱（Binning）是什么？

连续数值很难直接分析，所以我们把它切成区间，就像把成绩分成"优秀/良好/及格/不及格"。

例如，把长度差值按 20 个等宽区间划分：

| 差值区间 | 区间中心 | 该区间配对数 | 更长者的胜率 |
|---------|---------|------------|------------|
| (−∞, −3500] | −3500t | 很少 | 约 25% |
| ... | ... | ... | ... |
| (0, 400] | 200t | 很多 | 约 53% |
| ... | ... | ... | ... |
| (2618, 2992] | **2792t** | 782 | **74.3%** |
| ... | ... | ... | ... |

### 3.3 关键发现

**全量数据（78,970 对）**：

- 在所有方向明确的配对中，**62.2%** 的情况下，更长的那个模型获胜
- 长度差值与胜率的 Pearson 相关系数：**r = 0.927**（极强正相关！）
- 当 A 比 B 多约 **2,792 个 token** 时，A 的胜率达到峰值 **74.3%**

```
胜率随长度差值的变化趋势（示意）：

胜率
 75% ─────────────────────────────⊙──────
 70%                        ⊙
 65%                  ⊙
 60%         ⊙
 55%  ⊙
 50%─────────────────────────────────────── 基准（随机）
 45%  更短者赢的区间
       ←─────────┬─────────────────────────→ 长度差值 (tokens)
                 0      +1000   +2000   +3000
```

**按任务类型分层**：

| 任务 | 相关系数 r | 峰值胜率 | 最优差值 |
|------|-----------|---------|---------|
| 创意写作（CW）| **0.972** | **90.6%** | 2,787t |
| 指令遵循（IF）| 0.924 | 74.1% | 3,146t |
| 全量 | 0.927 | 74.3% | 2,792t |

> 创意写作中，长度偏好最强——当 A 比 B 多约 2,787 token 时，A 的胜率高达 **90.6%**！这提示创意写作任务中"内容越丰富越受欢迎"的规律更为突出。

### 3.4 相关系数（Pearson r）怎么理解？

Pearson 相关系数 $r$ 衡量两个变量之间的**线性关系强度**，取值 $[-1, +1]$：

| r 的值 | 含义 |
|--------|------|
| 接近 +1 | 强正相关（一个增大，另一个也增大）|
| 接近 0 | 几乎无线性关系 |
| 接近 −1 | 强负相关（一个增大，另一个减小）|
| 0.927 | 非常强的正相关 |

> **警告**：$r = 0.927$ 只说明"长度差值"和"胜率"**同步变化**，不能证明"因为更长所以更受欢迎"——也可能是因为质量好的模型恰好也写得更长。这正是第三阶段要解决的问题。

## 4. 格式偏好分析（C15）

### 4.1 什么是"格式"？

我们统计的三种格式标记（基于 Markdown 语法）：

| 格式类型 | Markdown 写法 | 渲染效果 |
|---------|-------------|---------|
| **标题**（header）| `## 小标题` | 大字体分节符 |
| **列表**（list）| `- 第一点` 或 `1. 第一步` | 项目列表 |
| **粗体**（bold）| `**重要内容**` | **加粗文字** |

提取方法：把模型回答的文本用正则表达式扫描，计算每种格式出现的次数。

### 4.2 格式存在性对胜率的影响

**最直观的问题**：有格式的回答 vs 无格式的回答，谁更容易获胜？

| 格式类型 | 有格式时胜率 | 无格式时胜率 | 胜率提升 |
|---------|-----------|-----------|---------|
| 含任意格式 | **51.5%** | 42.1% | **+9.4%** |
| 含标题 | **55.1%** | 46.3% | **+8.7%** |
| 含列表 | **51.9%** | 44.0% | **+8.0%** |
| 含粗体 | **53.1%** | 42.0% | **+11.1%** |

> **粗体**的提升效果最大（+11.1%）。没有任何格式的纯文字回答，胜率只有 42.1%，低于随机基准（50%）。

### 4.3 格式数量的影响——一个陷阱

如果画出"标题数量"vs 胜率的曲线，会发现最优标题数量是 **21 个**（胜率 0.594），最优列表数量是 **36 个**。

**但这个结论有一个致命问题！**

格式的数量和回复的长度**高度相关**：一个 3,000 词的回复，自然可以有更多标题和列表；一个 100 词的短回复，只能有 1-2 个。

```
问题示意：
  短回复（200t）：   1 个标题，  3 个列表 → 胜率低
  长回复（2000t）：  8 个标题，  25 个列表 → 胜率高

  结论：是格式多所以赢了？还是因为回复长所以格式多才赢？
```

这就是**统计学中的"混淆变量"（Confounding Variable）问题**：长度同时影响了"格式数量"和"获胜概率"，导致我们错误地把长度的功劳算在了格式上。

### 4.4 解决方案：格式密度

为了剥离长度的影响，我们引入**格式密度**这一指标：

$$\text{格式密度} = \frac{\text{格式出现次数}}{\text{回复 token 数}}$$

它回答的问题是：**平均每个 token，用了多少格式？**

| 格式密度指标 | 最优密度值 | 对应胜率 |
|------------|---------|---------|
| 标题密度 | 0.0020（每 500t 约 1 个标题）| 58.1% |
| 列表密度 | 0.0057（每 175t 约 1 个列表项）| 53.9% |
| 粗体密度 | 0.0509（每 20t 约 1 处粗体）| 56.2% |

这是一个"单位化"操作，和用"速度"（米/秒）代替"位移"（米）的道理相同。

### 4.5 格式组合分析

不同格式的**组合**效果如何？

| 格式组合 | 胜率 | 样本量 |
|---------|------|--------|
| 标题 + 列表 + 粗体（三种全有）| **55.7%** | 60,258 对 |
| 标题 + 粗体 | 51.3% | 1,807 对 |
| 列表 + 粗体 | 50.8% | 42,388 对 |
| 纯文字（无格式）| 42.1% | 25,097 对 |

> 三种格式全有的回答胜率最高（55.7%），但这同样可能与长度正相关（有三种格式的回答通常也更长）。

## 5. 探索性阶段的局限性

到目前为止，我们发现了两条清晰的规律：

- ✅ 更长的回答更容易获胜（相关系数 r = 0.927）
- ✅ 有格式的回答更容易获胜（+8%~+11% 的胜率提升）

但这些发现有一个共同的局限性：**我们只是在描述"现象"，还没有解释"原因"**。

### 5.1 相关 ≠ 因果

```
鸡冰淇淋销售量随气温升高而增加 ─────→ 吃冰淇淋导致溺水？
溺水事故数量也随气温升高而增加 ─┘

真实原因：气温（混淆变量）同时影响了两个变量
```

类似地：

```
强能力的模型 → 写出更长的回答 → 更容易获胜
强能力的模型 → 写出更格式化的回答 → 更容易获胜

我们观察到的"长度/格式 → 获胜"的规律，
有多少是"真正的长度/格式偏好"？
有多少是"强模型恰好更长"带来的假象？
```

这就是第三阶段（统计检验）和第四阶段（净效应分析）需要回答的问题。

### 5.2 子集分析：任务类型的重要性

不同任务类型下，偏好规律并不相同。以创意写作为例：

| 指标 | 创意写作（CW）| 指令遵循（IF）| 全量 |
|------|-------------|-------------|------|
| 有格式时胜率提升 | +6.0% | 更高 | +9.4% |
| 最优标题数 | **2 个** | 更多 | 21 个 |
| 长度–胜率相关 r | **0.972** | 0.924 | 0.927 |

创意写作中最优标题数只有 2 个（而非全量的 21 个）——因为堆砌大量标题的创意文章反而显得不自然，会被人类评估者扣分。这表明**格式的"最优量"因场景而异**。

## 6. 关键统计概念小结

| 概念 | 一句话解释 | 本研究中的应用 |
|------|-----------|-------------|
| **分箱（Binning）** | 把连续数值切成区间，用于计算组内均值/比例 | 把长度差值分 20 组，每组算胜率 |
| **相关系数（r）** | 两变量线性同步变化的强度，[-1, +1] | 长度差值 vs 胜率，r = 0.927 |
| **混淆变量** | 同时影响自变量和因变量的第三个变量 | 模型能力同时影响长度和胜率 |
| **格式密度** | 格式出现次数 ÷ 总 token 数 | 控制长度后的格式化程度 |
| **基准胜率** | 随机选择时的期望胜率（50%）| 用于判断偏好是否超出随机 |
| **分层分析** | 按某一变量的取值分组后分别分析 | 按 CW/IF/MATH/CODE 分别计算 |

## 7. 思考题

1. 在图表中，长度差值为 0 时（两个模型的回答一样长），胜率应该是多少？为什么？

2. 如果一个任务类型中，"格式密度"对胜率的影响为零，这意味着什么？（提示：结合格式密度的定义思考——控制长度后，纯格式还有没有额外价值？）

3. 全量数据的"长度差与胜率相关 r = 0.927"，但这个数值是计算的是**分箱后每组的平均胜率与分箱中心**之间的相关，而不是每对配对的 0/1 结果与差值的相关。这两种计算方式有什么区别？哪种更适合展示趋势？
