## 1. 这份总导读是做什么的

现有的 introduction 系列原本分成总索引和五个阶段，优点是便于分段讲解，但缺点也很明显：

- 如果是第一次进入项目，很容易在多个文件之间来回跳转
- 如果只是想快速建立全局地图，会先被局部细节打断
- 如果要给学生或老师做一次完整展示，往往还要自己再手工整合一遍

所以这份 `general_introduction.md` 的目标不是替代所有原文件，而是把它们合并成一份**可以独立阅读的总导读**。  
它保留原来的五阶段结构，但把工程入口、数据结构、方法逻辑、结果口径和教学重点放到同一条连续叙事里。

如果只想读一份 introduction 文件，优先读这一份。

## 2. 先用一句话理解整个项目

这项研究不是在简单讨论“人类是不是更喜欢长回答”，而是在追问：

**当人类在两个大语言模型输出之间做偏好比较时，他们会不会系统性奖励某些表层形式特征，例如更长、更多标题、更多粗体；如果会，这些信号又会怎样被 RLHF 或 DPO 这样的对齐流程继续放大。**

换句话说，本项目真正研究的是：

**偏好数据本身究竟在奖励什么。**

这也是它同时具有工程价值、统计价值和方法论价值的原因。

## 3. 当前版本最重要的研究口径

在进入工程细节之前，建议先记住当前终稿已经稳定下来的四条核心口径：

### 3.1 数据口径

- 原始整合数据共有 135634 条记录
- 经过清洗后保留 108171 条样本
- 排除 `tie` 和 `both_bad` 后，长度与偏好方向明确的有效配对为 78970 组
- 只保留 `evaluation_order = 1`，因为更高轮次会污染长度相关字段的解释口径

### 3.2 长度结论

- 全量样本中，胜者更长的比例为 62.21%
- 胜者相对败者的中位长度差为 +125 tokens
- 单侧 Wilcoxon 检验的 rank-biserial r 为 0.3362
- 控制混淆后，长度净效应仍保留，M3 OR = 1.3551
- 处理效应框架下，调整 OR = 1.9103，IPW ATE = 0.1489，匹配后 ATE = 0.1601

### 3.3 格式结论

- 标题密度和粗体密度是更稳定的独立格式信号
- 格式净效应模型中，标题 F3 OR = 1.0606，粗体 F3 OR = 1.0855
- 格式稳健性分析中，标题 IPW ATE = 0.0277，粗体 IPW ATE = 0.0246
- 列表密度只保留为敏感性变量，不作为正式主结论

### 3.4 机制结论

- SEM 主模型拟合较好，CFI 和 TLI 都接近 1，RMSEA = 0.0030
- 标题密度和粗体密度的直接效应 Bootstrap 区间排除 0
- 长度变量虽然在现象层和稳健性层都很稳，但其直接效应区间跨 0

因此，当前项目最稳妥的总结是：

**长度偏好是最稳健的经验发现，标题和粗体是更稳定的独立格式信号，列表不是稳定主结论，而长度在机制层面必须比标题和粗体更谨慎地解释。**

## 4. 先看工程全地图

如果把整个仓库看成一座研究工厂，它大致分成下面几层。

### 4.1 `Codes/`：研究流水线本身

这里放的是各阶段脚本，也就是“研究是怎么一步步推进的”。

- C01-C03：检查原始数据是否能作为可信研究对象
- C04-C11：逐字段摸底，确认哪些字段能进分析
- C12：清洗、过滤、结构重组，形成优化底表
- C13：按任务标签切出子集
- C14-C17：先做描述性分析和正式统计检验
- C18-C19：估计长度和格式的净效应
- C20-C22：做长度稳健性、格式稳健性和匹配诊断
- C23：用 SEM 解释机制路径

### 4.2 `Data/`：原料、中间产物和分析输入

这里放的是原始 parquet 分片、整合后的总表、优化后的分析表，以及各类子集文件。

最关键的两个数据节点是：

- `integrated_data.parquet`：把原始 7 个分片统一合并后的底表
- `optimized_data.parquet`：清洗并重组后、供后续分析统一消费的优化表

### 4.3 `Reports/`、`Tables/`、`Pictures/`：证据输出层

- `Reports/`：更适合给人读的自然语言结果说明
- `Tables/`：结构化统计结果
- `Pictures/`：图表与可视化结果

对课堂或答辩来说，这三层可以理解为：

- `Reports/` 负责解释
- `Tables/` 负责证据
- `Pictures/` 负责直观呈现

### 4.4 `References/`：方法教学与写作支撑层

这里放的是 introduction、方法导读、论文框架等配套材料。  
你现在读到的这份总导读，就属于这一层。

## 5. 第一阶段：认识课题、数据与工程入口

### 5.1 这一阶段到底在解决什么问题

第一阶段的目标不是做统计，而是先回答三个更基础的问题：

1. 这项研究到底在问什么。
2. 一行数据到底代表什么。
3. 原始数据结构本身是否值得相信。

如果这三件事不先讲清，后面的任何统计结果都可能建立在错误的研究对象上。

### 5.2 这项研究到底在问什么

本项目研究的是：

**当人类在两个模型回答之间做选择时，他们会不会系统性偏好某些表层形式特征。**

这里的“表层形式特征”主要包括：

- 更长的回答
- 更清晰的标题结构
- 更多粗体强调
- 更明显的列表组织

这个问题看起来像用户体验问题，但本质上是对齐研究问题。  
因为如果人类偏好标签本身包含了这些启发式偏差，那么用这些标签训练出来的奖励模型和对齐模型，就会把“看起来像好答案”的外观特征也学进去。

### 5.3 一行数据代表什么

这份数据不是普通的“一题一答”表，而是一个带层级的对话偏好数据集。  
最基本的层级关系是：

`Session -> Order -> Turn`

- `Session`：一次完整评测会话
- `Order`：同一会话中的一次评价顺序
- `Turn`：这一轮对话内部的问答轮次

初学者最容易误判的地方是：

**数据表中的一行不是一个完整 Session，而是一条 Order 级别的评价记录。**

这也是为什么后面必须检查 `evaluation_order`，并最终只保留首轮评价。

### 5.4 为什么工程第一步不是统计，而是 C01-C03

因为本项目坚持一个基本研究原则：

**先确认研究对象，再进入分析。**

这一阶段的三个脚本分别做了三件事：

- C01：检查同一个会话是否被拆散到不同分片里
- C02：合并 7 个原始 parquet 分片，恢复统一数据入口
- C03：检查 `evaluation_order` 是否连续、一致，确认会话结构本身没有明显异常

用教学语言来讲，这一步做的是：

**先确认“我们到底在分析什么”，而不是一上来就让模型和统计量接管叙事。**

### 5.5 第一阶段真正教给学生的东西

这一阶段最重要的不是三段代码，而是三条研究原则：

- 先确认研究单位
- 先确认数据结构完整性
- 先做审计，再谈分析

学生学完这一阶段后，至少应该能回答：

1. 这个项目研究的偏差是什么。
2. 一行数据代表的是 Session 还是 Order。
3. 为什么原始数据不能直接拿来做结论。

## 6. 第二阶段：从原始数据到可分析数据

### 6.1 第二阶段在解决什么问题

经过第一阶段，我们已经知道数据“能读、能合并、结构大体可信”。  
但这并不等于它已经可以直接进入统计分析。

原始数据仍然存在四类问题：

- 字段嵌套很深
- 字段含义不完全统一
- 一些字段对分析没有帮助
- 一些标签还不能直接拿来做分层或建模

所以第二阶段真正要做的是：

**把复杂原始对象加工成分析友好型数据表。**

### 6.2 为什么要先做 C04-C11 的逐字段摸底

这一组 `touch_` 脚本虽然不直接产出显著性结果，却非常关键。  
它们的作用像体检：先回答“这个字段能不能信、能不能用”。

这一组脚本主要在确认：

- Session 和 Order 的分布长什么样
- 对话轮次会不会影响长度解释
- 哪些模型出现得更多、胜率更高
- 角色与内容结构是否适合扁平化
- token 汇总字段的定义是否一致
- 任务标签能不能被安全转为布尔变量

### 6.3 C10 为什么是整个项目的关键节点之一

C10 发现了一个如果不处理就会把后续长度研究全部带偏的问题：

- `sum_assistant_a_tokens` 和 `sum_assistant_b_tokens` 代表当前比较对象的回答长度
- 但 `sum_user_tokens` 在 `evaluation_order > 1` 时会累积更早轮次的会话历史

这意味着：

**如果把更高轮次的评价记录混进来，那么某些长度相关变量就不再只代表当前这次比较，而会掺入之前轮次的历史。**

所以只保留 `evaluation_order = 1`，不是随意裁剪样本，而是为了保证分析口径统一。

### 6.4 为什么 C12 是整个工程的转折点

C12 标志着项目从“原始数据世界”正式进入“分析数据世界”。  
它做的事情不只是清洗，而是同时完成了：

1. 过滤异常样本
2. 固定关键口径
3. 重组嵌套结构
4. 保留分析所需原料字段
5. 写出优化后的统一输入表

当前终稿口径下，最关键的清洗结果是：

- 原始整合记录 135634 条
- 清洗后保留 108171 条
- 其中绝大多数被剔除的样本都和更高轮次的 `evaluation_order` 或其他异常结构有关

### 6.5 结构重组到底是什么意思

当前项目并没有把所有信息暴力摊平成大量顶层列，而是采用了更稳定的 nested schema：

- `conv_a`、`conv_b`、`conv_user`：保存对话结构
- `metadata_a`、`metadata_b`、`metadata_user`：保存长度和格式原料
- `category_tag`：统一保存任务类型和质量维度标签

这种设计体现的是一个很值得教给学生的工程思想：

**分析数据结构不等于“列越多越好”，而是要尽量稳定、可复用、可按需展开。**

当前版本中，后续脚本统一通过 `Codes/accessor.py` 从这套 nested schema 中展开所需变量，而不是各自重新维护路径和字段转发逻辑。

### 6.6 为什么 C13 还要继续划分子集

有了总表以后，项目并没有直接只在全量样本上分析，而是继续切出 20 个子集。  
原因很简单：

**同一个偏好规律，在不同任务类型下可能强弱不同，甚至解释方式不同。**

C13 主要切出两类子集：

- 单类子集：只要包含某类任务就纳入，允许重叠
- 纯净分区：按四个布尔任务标签的真值组合切出 16 个互斥子集

这一阶段最适合教学生的，是“总体结论和分层结论未必一致”。

### 6.7 第二阶段真正教会学生什么

第二阶段最重要的不是清洗命令本身，而是三种统计前思维：

- 数据口径必须统一
- 字段含义必须先被审计
- 研究者要主动构造分析对象，而不是被动接受原始字段

如果这一步没讲清，后面的统计方法会显得像在“直接处理现成数据”；而实际上，真正的研究是先把数据制造成可以解释的对象。

## 7. 第三阶段：从图形直觉到统计检验

### 7.1 为什么到了这一步才开始讲统计

因为前两个阶段解决的都是前置问题：

- 数据结构是否可信
- 变量口径是否统一
- 子集划分是否清楚

只有这些条件满足后，我们才有资格讨论：

**长度偏好和格式偏好是否真的存在。**

这一阶段可以浓缩成一句话：

**先用图表看见现象，再用统计检验判断它靠不靠谱。**

### 7.2 C14 和 C15 在做什么

这两个脚本做的是“现象层”工作。

#### C14：长度偏好的描述性分析

它主要做了：

- 剔除 `tie` 和 `both_bad`
- 构造有方向的长度差值
- 观察不同长度差对应的胜率变化
- 比较全量样本与不同任务子集中的长度偏好图像

它回答的问题是：

**回答更长时，人类偏好获胜率会不会更高。**

#### C15：格式偏好的描述性分析

它把格式拆成三类：

- 标题
- 列表
- 粗体

并且不只看“有没有”，还看：

- 绝对计数
- 格式组合
- 格式密度，也就是 `count / (tokens + 1)`

它回答的问题是：

**除了长度以外，回答的组织形式会不会影响选择。**

### 7.3 为什么图看起来很明显，还不能直接下结论

这是第三阶段最关键的教学点之一：

**看起来明显，不等于统计上成立。**

图上的趋势可能来自：

- 真正稳定的规律
- 样本量波动
- 极端值拖动
- 某些子集恰好偏多

所以第三阶段要完成的，不是把图画得更漂亮，而是把“观察”升级成“证据”。

### 7.4 为什么本项目主要用 Wilcoxon、Bonferroni 和 Bootstrap

这里的配对差值数据通常有这些特点：

- 偏态
- 长尾
- 极端值较多
- 中位数往往比均值更有解释力

因此，本项目在正式检验阶段主要使用：

- Wilcoxon 符号秩检验：适合配对差值、对分布要求更稳健
- Bonferroni 校正：防止多重检验导致碰巧显著
- Bootstrap 置信区间：给出不确定性范围，而不是只报点估计

对零基础学生来说，这一阶段最重要的理解是：

- `p` 值回答“像不像随机波动”
- 效应量回答“这个差异到底有多强”
- 置信区间回答“这个结论稳不稳”

### 7.5 C16 和 C17 当前对应的关键结果

#### 长度偏好

在正式检验阶段，当前终稿对应的全量样本结果为：

- 有效配对数 78970
- 胜者更长比例 62.21%
- 中位长度差 +125 tokens
- rank-biserial r = 0.3362

这说明长度偏好不是局部样本偶然现象，而是一个在全量样本上稳定存在的中等强度现象。

#### 格式偏好

格式偏好则需要更谨慎地讲：

- 标题、列表、粗体在绝对计数层面都可能表现出正向信号
- 但一旦切换到密度尺度，列表的独立性就明显减弱
- 标题和粗体比列表更接近稳健格式信号

这一步很适合教学生：

**同一个“格式偏好”问题，不能只用一个指标回答。**

### 7.6 第三阶段真正教会学生什么

这一阶段最该让学生记住五句话：

1. 图表是观察，不是最终证据。
2. 显著性不等于作用很大。
3. 同一个问题最好从多个统计视角交叉验证。
4. 变量显著，不代表它已经是独立原因。
5. 正式检验的目标不是制造数字，而是判断观察是否足够可靠。

到这里，研究已经从“现象描述”推进到“统计确认”。

## 8. 第四阶段：为什么相关不等于因果

### 8.1 为什么第三阶段之后还不能停

到了第三阶段，学生最容易产生一个自然判断：

“既然统计显著了，那不就说明更长或更有格式的回答更容易赢吗？”

这正是第四阶段必须出现的原因。  
因为在这里，项目要正式引入一个研究里最关键的思想：

**相关不等于因果。**

例如，下面这种情况完全可能同时发生：

- 更强的模型本来就更容易获胜
- 更强的模型也更可能写得更长、更爱用格式

这就意味着，我们在现象层观察到的优势，有一部分可能只是“模型能力更强”的外溢结果。

### 8.2 什么叫混淆变量

最适合讲给零基础学生的解释是：

**一个变量既会影响原因，也会影响结果，于是它让两者看起来关系特别强。**

在本项目里，典型混淆因素包括：

- 模型能力差异
- 任务类型差异
- 问题复杂度和知识要求
- 模型风格习惯

所以第四阶段真正要问的是：

**把这些背景因素按住以后，长度和格式还剩下多少独立作用。**

### 8.3 C18-C19：净效应分析在做什么

这一阶段首先用嵌套逻辑回归，逐层加入控制变量，观察核心效应会缩小多少。  
长度系列模型从 M0 到 M3，格式系列模型从 F0 到 F3，本质上都在做一件事：

**先看粗效应，再看控制混淆后还能剩多少。**

当前终稿口径下，最关键的结果是：

- 长度净效应在 M3 中仍保留，OR = 1.3551
- 标题密度和粗体密度在格式净效应模型中分别保留 F3 OR = 1.0606 和 1.0855
- 列表密度在多框架中都不够稳定，因此只保留为敏感性变量

这一步最值得教学生的是：

**显著现象经过控制后仍然存在，比“直接显著”更有说服力。**

### 8.4 为什么还要继续做长度稳健性分析

如果一个结论只靠一种统计方法支撑，它的可信度仍然有限。  
因此项目继续把“更长”转换为处理变量，进入处理效应视角。

长度稳健性分析要回答的是：

**如果把处理组和对照组在背景条件上尽量调平，更长这件事还会不会带来胜率提升。**

当前终稿里，最关键的两个数字是：

- 调整 OR = 1.9103
- IPW ATE = 0.1489

它们说明，长度优势并不只是相关层面的幻象，而是在处理效应框架下也能被重复识别。

### 8.5 为什么格式也要单独做稳健性分析

如果只做净效应回归，学生很容易误以为“格式只是长度分析的附带部分”。  
实际上，项目把标题、粗体和列表分别放进格式稳健性分析中，就是为了回答：

**在长度和其他格式变量尽量按住以后，这一种格式密度还有没有独立作用。**

当前终稿中的关键结果为：

- 标题调整 OR = 1.1317，IPW ATE = 0.0277
- 粗体调整 OR = 1.1060，IPW ATE = 0.0246
- 列表调整 OR = 1.0265，IPW ATE = 0.0016，区间跨 0

这一步最适合教学生的一点是：

**不同格式变量不能被打包成一个笼统的“格式优势”。**

### 8.6 为什么最后还要做匹配诊断

因为项目不只想知道“处理效应看起来存在”，还想知道：

**这种处理效应是不是建立在足够可比的样本上。**

匹配诊断做的是另一种验证：为处理组尽量找到背景条件相似的对照样本，再看结论是否仍然成立。

当前版本最关键的匹配结果是：

- 匹配对数 39237
- mean |SMD| 从 0.2502 降到 0.0597
- 匹配后 ATE = 0.1601

这意味着即使把处理组和对照组尽量配平，长度优势仍然没有消失。

### 8.7 第四阶段真正教会学生什么

第四阶段最重要的不是某个高级模型，而是研究可信度的建构逻辑：

- 不是一个回归显著就够了
- 不是换个方法简单重复一遍
- 而是要不断追问：结论会不会只是混淆、会不会依赖某个估计框架、会不会建立在不可比样本上

学完这一阶段，学生应该真正理解：

**好研究要经得起反复拷问。**

## 9. 第五阶段：从净效应走向机制解释

### 9.1 为什么到了最后还要做 SEM

前四个阶段已经回答了很多问题：

- 长度偏好存在吗
- 格式偏好存在吗
- 控制混淆后还在吗
- 换方法后还稳吗

自然就会出现一个更进一步的问题：

**这些因素究竟是怎样共同作用于偏好形成过程的。**

例如：

- 模型能力差异会不会先让回答变长，再提高胜率
- 模型格式倾向会不会通过标题或粗体进入偏好判断
- 长度和格式是在互相替代，还是并列发挥作用

这正是 SEM 出场的原因。

### 9.2 怎样把 SEM 讲给零基础学生听

最适合的讲法不是从矩阵和协方差开始，而是直接告诉学生：

**SEM 是把多个回归关系放进一张路径图里，同时看谁影响谁、通过哪条路影响。**

换句话说，它在做一张“变量关系地图”。

### 9.3 本项目里的 SEM 并不神秘

当前项目使用的是观测变量 SEM。  
它复用前面阶段已经稳定下来的配对差异变量，而不是凭空再造一套全新指标。

在这套模型里：

- 结果变量是 `winner_a`
- 核心中介变量是 `token_diff_ab`、`header_density_diff`、`bold_density_diff`
- 扩展模型把 `list_density_diff` 作为敏感性变量加入
- 外生变量包括 `ability_diff`、`verbosity_diff`、`format_tendency_diff`、`user_tokens`、`turns` 和任务、质量维度布尔变量

这一步最值得学生理解的是：

**机制模型不是脱离前文单独出现的，它建立在前面已经打磨好的变量体系之上。**

### 9.4 什么叫直接效应、间接效应和总效应

这是第五阶段最适合讲清楚的基本概念。

- 直接效应：变量不经过别的桥梁，直接影响结果
- 间接效应：变量先影响中介，再由中介影响结果
- 总效应：直接效应与间接效应之和

在本项目里，一个非常适合教学的表达是：

**有些变量不是自己直接决定输赢，而是先改变回答形态，再通过回答形态影响偏好。**

### 9.5 这一阶段最值得讲的结果

#### 模型整体拟合良好

当前终稿中，主模型的：

- CFI 接近 1
- TLI 接近 1
- RMSEA = 0.0030

这说明路径图整体上和数据贴合得很好。  
但这里也特别适合提醒学生：

**模型整体拟合好，不等于每一条路径都能被过强解释。**

#### 标题与粗体拥有更清晰的直接路径证据

当前稳定口径是：

- 标题密度的直接效应 Bootstrap 区间排除 0
- 粗体密度的直接效应 Bootstrap 区间排除 0

这意味着标题和粗体更接近机制层面的直接结构信号。

#### 长度是最稳健的经验发现，但不是最简单的直接路径

长度在前面的现象检验、净效应分析、IPW 和匹配诊断里都非常稳定。  
但进入 SEM 之后，它的直接效应区间跨 0。

这一步最值得教学生区分的，是下面两件事：

- “统计上稳健相关”
- “机制上已经被清楚识别为直接路径”

长度显然属于前者，但在当前项目中不能被简单讲成后者。

#### 列表不是主结论变量

列表密度在扩展模型中只保留为敏感性变量。  
它没有像标题和粗体那样，形成稳定而清晰的直接路径证据。

### 9.6 第五阶段真正教会学生什么

第五阶段最重要的不是让学生背 SEM 术语，而是让他们理解：

**高质量研究不只会发现差异，还会尝试解释差异从哪里来、通过什么发生、在多大程度上成立。**

到这里，整条研究链条才真正闭合。

## 10. 如果只给学生讲一遍，推荐怎么讲

### 10.1 最推荐的讲解顺序

如果要做一次课堂展示或组会汇报，建议按下面顺序推进：

1. 先讲研究问题：偏好数据可能奖励表层形式，而不只是内容质量。
2. 再讲数据结构：一行记录是 Order，不是 Session。
3. 然后讲为什么必须先做数据审计，再做清洗和子集构造。
4. 接着讲现象层：先用图表看见长度与格式偏好，再用正式检验确认它们。
5. 再讲净效应与稳健性：证明结论不是纯混淆、不是只在一种方法里显著。
6. 最后讲机制层：标题和粗体更像直接结构信号，长度虽然稳健，但机制更复杂。

### 10.2 如果只能用一句话总结整条研究路线

最适合写在 PPT 或课堂板书上的一句话是：

**先确认数据可信，再制造统一口径的分析字段，再观察现象，再做统计检验，再控制混淆、验证稳健性与可比性，最后用 SEM 解释偏好是如何形成的。**

### 10.3 如果只能用五句话讲完整个项目

1. 本项目研究的是人类偏好数据会不会系统性奖励长度和格式等表层特征。
2. 在做任何统计之前，项目先确认了研究单位和数据结构是否可信。
3. 在统一口径后，研究发现长度偏好稳定存在，标题和粗体也具有较强格式信号。
4. 控制混淆并做多框架稳健性分析后，长度、标题和粗体仍保留独立作用，而列表不稳定。
5. 机制分析进一步表明，标题和粗体更接近直接结构信号，长度则是稳健相关但机制更复杂的综合变量。

## 12. 最后的总总结

这项研究最值得学生学到的，不只是某个统计方法，也不只是某几个显著结果，而是一整套研究推进方式：

- 不把原始数据默认当成可信对象
- 不把图上趋势直接当成结论
- 不把相关关系直接当成因果解释
- 不把单一方法的显著结果当成最终答案
- 不把机制模型当成对前文无依赖的“高级装饰”

真正严谨的研究，是沿着下面这条路线一步步推进的：

**先审计对象，再统一口径，再识别现象，再控制混淆，再验证稳健性，最后解释机制。**

这也是这份总导读希望帮助读者真正建立起来的研究思维。
