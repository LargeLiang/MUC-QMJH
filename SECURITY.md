# 安全与合规说明

本仓库主要用于研究代码、方法说明和结果复现，不面向生产系统。尽管如此，以下问题仍建议按安全与合规事件处理：

- 可能导致敏感数据泄露的脚本或配置
- 误上传原始数据、受限数据或第三方受版权保护材料
- 会误导使用者公开再分发受限数据的文档表述
- 明显影响结果可信度的高风险实现缺陷

## 1. 建议披露方式

请不要在公开 Issue 中直接贴出：

- 受限数据样本
- 凭据、私钥、令牌或路径中的个人信息
- 可直接导致数据外泄的细节

如仓库已启用 GitHub Private Vulnerability Reporting，优先使用私有通道。若尚未启用，请联系项目维护者的私下沟通渠道。

## 2. 当前限制

本仓库目前未在公开文件中提供专用安全邮箱。正式公开前，维护者应补充：

- 安全联系邮箱
- 响应时间承诺
- 对数据合规问题的内部处理人

## 3. 响应目标

建议维护者按以下节奏处理：

- 2 个工作日内确认收到
- 7 个工作日内给出初步判断
- 对数据和许可问题优先于一般 bug 处理

## 4. 支持范围

当前支持范围包括：

- 主分析链脚本 C01-C22
- README 与许可说明
- 结果文件引用和公开边界问题

不承诺提供生产环境级别的安全支持、长期补丁维护或外部部署支持。
