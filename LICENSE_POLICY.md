# 仓库许可与数据边界说明

本文件用于说明本仓库中不同类型内容的许可边界。它是对根目录 LICENSE 的补充说明，不替代上游数据许可，也不构成法律意见。

## 1. 代码许可

除另有说明外，本仓库中的自研源代码按 Apache License 2.0 发布。

默认适用范围包括：

- Codes/
- check_nb.py
- 未来新增的自研 Python、R、Shell、Notebook 调度代码

Apache-2.0 适合作为科研代码仓库的默认许可，原因是：

- 允许再使用、再分发和改写
- 对贡献者权利与责任边界说明较完整
- 包含专利许可条款，较 MIT 更适合长期协作仓库

## 2. 文档与教学材料许可

除另有说明外，本仓库中的自研说明文档、教学材料、方法介绍和自撰写研究说明文本按 CC BY 4.0 共享。

默认适用范围包括：

- README.md
- References/
- CHANGELOG.md
- CONTRIBUTING.md
- SECURITY.md
- CODE_OF_CONDUCT.md

允许转载、改编和引用，但应保留署名并注明修改。

## 3. 研究输出文件说明

本仓库中的自生成研究输出，例如：

- Reports/
- Tables/
- Pictures/

在不违反上游数据许可的前提下，可按 CC BY 4.0 引用和转载其表达性内容。该说明不授予对底层第三方数据的额外权利，也不表示可绕开上游数据集的许可条件。

## 4. 数据文件边界

Data/ 下的原始数据、缓存数据、派生数据和任何可回溯至第三方数据集的文件，不受根目录 LICENSE 的统一授权。

应遵循以下规则：

- 原始数据的著作权和许可归上游数据提供方所有
- 本仓库不对第三方数据授予任何额外再许可
- 使用者应自行阅读并遵守上游数据集条款
- 如需公开再分发派生数据，应先确认其是否满足原数据许可要求和所在机构规范

## 5. 第三方材料

若仓库中包含第三方论文、截图、图标、引用片段或其他受限材料，应以原权利声明为准。除非文件中另有明确说明，本仓库不对该类第三方内容作额外授权。

## 6. 仓库公开使用建议

对于公开 GitHub 仓库，建议遵循以下原则：

- 公开代码与说明文档
- 不直接上传未经确认可再分发的原始数据
- 对研究输出保持引用要求和数据来源声明
- 在正式公开前，再次确认所在学校、课题组和项目负责人的公开政策

## 7. 许可方案摘要

本仓库当前采用的默认方案为：

- 代码：Apache-2.0
- 文档：CC BY 4.0
- 数据：不作统一再授权，以上游许可为准

如未来项目所有者决定调整公开策略，应同步更新本文件、README.md 和相关仓库元数据。
