## 一、课题定位与当前进展概览

本课题以 LMArena 人类偏好数据为对象，关注的不是“长回答是否一定更好”，而是更具体的问题：**当用户在两个大语言模型输出之间做选择时，长度、标题、列表、粗体等形式特征是否会系统性影响偏好判断；这些影响在控制任务属性、问题维度和模型差异后是否仍然成立；它们又通过什么路径进入最终偏好。**

围绕这一问题，研究主线已经形成较完整的五段式链条：

1. 先审计原始数据结构与研究单位是否可信。
2. 再将复杂嵌套数据清洗为统一口径的分析对象。
3. 用描述统计和正式检验确认偏好现象是否存在。
4. 用控制、加权与匹配方法检验净效应和稳健性。
5. 用结构方程模型解释偏好形成机制。

截至目前，主分析链 C01-C22 已完成并同步到当前仓库实现；结果文件、导读文档、方法文档和 README 也已对齐。研究重点已由“把分析链跑通”转向“压缩结论、整理论文表达与规范化交付”。

## 三、研究推进与阶段成果

### 3.1 第一阶段：数据审计与研究单位确认

对应脚本：C01-C03。

这一阶段解决的是“研究对象是否可信”，而不是“结果是否显著”。主要结论有三点：

1. 同一 evaluation_session_id 确实会跨 parquet 分片出现，因此原始分片不能独立分析，必须先合并。
2. 合并后数据规模为 135,634 × 14，为后续全部分析提供统一入口。
3. 同一 Session 内 evaluation_order 存在缺失和重复，因此高阶 order 不能直接视为与首轮完全同口径的独立样本。

这一步明确了原始数据的真实层级是 Session → Order → Turn，也为后续过滤 evaluation_order > 1 提供了对象层面的理由。

### 3.2 第二阶段：字段摸底与清洗依据建立

对应脚本：C04-C11。

这一阶段的任务是逐字段理解原始数据的结构、分布和异常，核心收获包括：

- role 标签体系非常干净，仅有 user 和 assistant 两类。
- content 基本为纯文本结构，多模态字段几乎不参与当前研究。
- category_tag 中已包含任务类别与七个质量维度，可支持后续分层与控制。
- 最关键的是，C10 证明 sum_user_tokens 在 evaluation_order > 1 时混入了会话历史，而非仅代表当前评价对象。

因此，“只保留 evaluation_order = 1”并不是经验性删减，而是为了保证长度相关变量口径一致。这是本课题最重要的一条清洗依据。

### 3.3 第三阶段：统一分析主表与子集体系

对应脚本：C12-C13。

C12 在 135,634 条整合记录中最终保留 108,171 条，过滤 27,463 条，保留率为 79.75%。过滤主因是 evaluation_order > 1 带来的 token 污染；其余原因包括空内容、语言异常和关键标签缺失。

更重要的是，C12 不是简单“删行”，而是把原始复杂结构重组为统一的 nested schema：

- 对话内容被整理为 conv_a、conv_b、conv_user。
- token 与格式统计被整理为 metadata_a、metadata_b、metadata_user。
- 任务标签与质量维度被整理为 category_tag 与 criteria。

C13 在此基础上继续生成 20 个任务子集，包括 4 个允许重叠的单类子集和 16 个互斥纯净分区。这样，后续研究既能做宏观比较，也能做更严格的分层分析。

### 3.4 第四阶段：描述统计与偏好存在性检验

对应脚本：C14-C17。

这一阶段回答的是“偏好现象是否存在”。当前可汇总为以下几点：

1. C14 的描述性分析显示，长度差值与胜率呈显著正相关，且这种趋势在多个任务子集中重复出现。
2. C15 的描述性分析显示，带有结构化格式的回答整体更容易获胜，但不同格式类型并不完全等价。
3. C16 在 17 个配置子集中完成长度 Wilcoxon 检验，其中 13 个达到最小样本量要求，12 个在 Bonferroni 校正后仍显著；全量样本中胜者更长的比例约为 62.21%，中位长度差约为 +125 tokens。
4. C17 进一步将格式分析从“计数差”推进到“密度差”。结果表明，标题与粗体在控制长度后仍具有更稳定的独立信号，而列表密度不再稳定显著。

这一阶段的意义在于：偏好现象已经从“图上看得出来”推进到了“统计上可以检验和比较”。

### 3.5 第五阶段：净效应、稳健性与机制解释

对应脚本：C18-C22。

这一阶段解决的是“这些偏好到底有多少是真效应，有多少是混淆放大的表观相关”。主要结果如下：

- C18 表明模型能力与平均长度、平均格式数确实相关，Spearman 相关分别为 0.7171 和 0.5318，说明模型层混淆必须显式控制。全量样本中，长度粗效应 OR 从 1.9045 衰减到 M3 的 1.3551；标题密度与粗体密度在 F3 中的净 OR 分别为 1.0606 和 1.0855，而列表密度仅为 1.0249，且被识别为“主要混淆”。
- C19 从处理效应视角重新检验长度优势。全量样本调整 OR 为 1.9103，IPW ATE 为 0.1489，95% CI 为 [0.1351, 0.1614]，说明长度优势在更严格的稳健性框架下仍然存在。
- C20 在控制长度差及其他格式差的前提下，得到标题密度调整 OR 1.1317、IPW ATE 0.0277，粗体密度调整 OR 1.1060、IPW ATE 0.0246；列表密度仅保留为敏感性分析变量，不写入主结论。
- C21 通过倾向得分匹配将全量样本的 mean|SMD| 从 0.2502 降到 0.0597，匹配后 ATE 仍为 0.1601，说明长度优势并不依赖原始样本失衡。
- C22 的主模型拟合优度很高，CFI = 0.9998，TLI = 0.9998，RMSEA = 0.0030。Bootstrap 结果显示：标题密度与粗体密度的直接效应置信区间均排除 0，而长度直接效应的置信区间跨 0。这提示长度虽然在多种模型中表现为稳健优势，但在机制层面更可能同时承载了冗长度、模型风格和表达组织等更复杂的代理信息。

至此，研究链条已经从“识别偏好现象”推进到“分解偏好来源并尝试解释机制”。

## 四、当前阶段性结论

基于现有脚本、结果文件与说明文档，可以将当前结论压缩为四点：

1. **数据口径已经统一。** 当前分析建立在 108,171 × 14 的 nested schema 主表之上，evaluation_order 污染问题已被明确识别并处理，后续统计不再建立在口径不一致的数据对象上。
2. **长度偏好稳定存在，但并非纯粹的“长度本身”效应。** 长回答在描述统计、Wilcoxon、调整 Logit、IPW 和匹配诊断中都表现出稳定优势；但 C18 与 C22 同时表明，这一优势受到模型能力、冗长度风格及其他代理变量的明显影响。
3. **格式偏好内部存在分化。** 标题和粗体在多轮控制后仍保留独立正向效应，列表则更像长度与风格的伴生指标，不宜作为核心结论单独强调。
4. **本课题已从现象识别推进到机制解释。** 当前最稳妥的表达不是“用户偏好长回答”，而是“人类偏好数据中存在可量化的形式偏差，其中长度、标题和粗体分别以不同方式进入最终偏好判断”。

这也意味着，本课题的成果已经可以支撑论文主体写作，而不再只是分散的工程实验结果。

## 六、个人学习与能力提升

本阶段最直接的收获，不只是脚本数量增加，而是研究方式发生了变化。

### 6.1 从“直接做统计”转向“先审计数据”

通过 C01-C03 与 C10，我对“研究单位”和“变量口径”有了更强的敏感性。现在会先确认数据结构是否可信，再讨论方法选择，而不是先画图或先做检验。

### 6.2 从“看 p 值”转向“理解效应、混淆与稳健性”

通过 C16-C22，我对显著性、效应量、混淆控制、处理效应估计和机制路径之间的区别有了更系统的理解。研究不再停留在“有没有显著”，而是进一步追问“效应有多大、来自哪里、换方法后还是否成立”。

### 6.3 从“写脚本”转向“搭建研究工程”

在整理 nested schema、共享访问层、统一结果编号和文档体系的过程中，我对复现性、可维护性和规范交付的重要性有了更具体的认识。现在更关注脚本职责边界、结果落盘规则和文档是否足以支撑继续维护。

### 6.4 从“完成分析”转向“能讲清分析”

在同步整理 introduction、method_list、paper_framework 和本报告的过程中，我对如何把代码逻辑、统计逻辑和论文叙事连接起来有了更清楚的把握。这对后续论文写作和答辩表达都很重要。
