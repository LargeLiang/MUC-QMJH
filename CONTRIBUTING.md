# 贡献指南

感谢你关注本仓库。该仓库目前以科研复现、研究记录和教学辅助为主要目标，因此贡献应优先服务于可复现性、可审阅性和合法合规性。

## 1. 开始前请先确认

- 不要提交未经许可可公开的原始数据、派生数据或第三方材料
- 不要改动已经作为正式结论引用的结果文件，除非同步说明原因和影响范围
- 不要在同一个提交中混合“方法变更”和“大量格式化清理”

## 2. 建议贡献范围

欢迎以下类型的贡献：

- 修复脚本错误或路径问题
- 改进复现说明、README 或方法文档
- 补充测试、环境说明和依赖约束
- 对报告中的数字或表格引用做一致性修正
- 在不改变研究结论口径的前提下改进绘图、日志和异常处理

## 3. 不建议直接提交的内容

- 未经核验的结论性改写
- 仅为了“更漂亮”而大范围改名或重排文件
- 未获确认的数据再分发
- 无说明地替换已归档表格、图片或报告

## 4. 代码风格要求

本项目优先遵循现有仓库风格：

- 路径统一使用 pathlib.Path
- 函数参数优先支持 root、file_path、data_dir、report_dir 这类可选路径注入
- Docstring 使用中文
- 类型注解尽量完整
- 输出报告使用统一分隔线和“任务完成！”收尾

## 5. 提交 Issue 前

提交 Issue 时，请尽量说明：

- 你运行的脚本名称
- 使用的数据文件或子集
- Python 版本与关键依赖版本
- 报错信息或结果差异截图
- 是否影响 README、报告结论或论文引用

## 6. 提交 Pull Request 前

PR 说明中建议包含：

- 变更目的
- 影响到的脚本或文档
- 是否影响现有结果
- 是否需要重新生成 Reports、Tables、Pictures 中的产物
- 已完成的验证步骤

## 7. 结果文件变更规则

若 PR 会影响以下目录，请在说明中显式标注：

- Reports/
- Tables/
- Pictures/

并说明这些变更是：

- Bug 修复导致的结果纠正
- 方法变更导致的结果更新
- 仅文案或格式修正

## 8. 许可与数据边界

贡献代码默认按 Apache-2.0 合并。文档和说明文本默认按 CC BY 4.0 共享。数据边界请参见 [LICENSE_POLICY.md](LICENSE_POLICY.md)。

## 9. 维护者补充项

公开发布前，建议维护者补充：

- GitHub 仓库所有者链接
- 联系邮箱
- 机构署名的正式英文写法
