## 二、当前仍需注意的残余风险

### 1. Notebook 内核与项目解释器不是同一个对象

这不是结果层风险，而是交互体验层风险。

影响：

- Notebook 内直接 `import pandas` 的行为仍可能落在 base 环境
- 因此 C00 继续采用“外部项目解释器跑脚本”的策略是必要的

建议：

- 课堂展示与正式复现都优先使用 C00 内置的 `run_project_script()`
- 不建议在 C00 中大量新增依赖 notebook 内核环境的分析代码

### 2. SEM 图当前依赖回退绘图

当前 C22 可稳定生成路径图，但在 graphviz 缺失时会回退到内置 networkx 图。

影响：

- 不影响结论
- 但图形美观度和论文级排版质量略弱

建议：

- 若后续进入论文定稿阶段，可补装 graphviz，重新导出高质量路径图

### 3. C20 与 C22 在 Notebook 中默认使用 fast mode

这是为了课堂演示速度，不是为了替代正式复现实验口径。

影响：

- Notebook 默认 bootstrap 次数较低
- 正式论文图表与正文数值仍应以 full mode 或独立脚本默认参数为准

建议：

- 课堂演示保持 `FAST_MODE = True`
- 论文复现切换到 `FAST_MODE = False`

## 三、研究拓展方向评估

### 方向 A：英语子集与多语言稳健性

**价值**：高  
**优先级**：P1

原因：

- 当前数据包含多语言，语言分布可能同时影响长度、格式和偏好判准
- 英语子集更接近公开 LLM benchmark 语境，也更利于对外发表时解释

建议实施：

1. 在 C13 新增 `lang_en_bool` 或英语子集导出
2. 对 C16-C22 复跑英语子集
3. 将“全量结果”和“英语结果”并排呈现为主分析 + 稳健性附录

### 方向 B：用户层/会话层相关性的稳健标准误

**价值**：高  
**优先级**：P1

原因：

- 当前一行视为一个独立配对
- 但不同 session 或同类 prompt 之间仍可能存在相关性

建议实施：

1. 为 C18-C20 加入 cluster-robust standard errors
2. 或考虑 mixed-effects logit / GEE 方案
3. 将“独立配对假设”的敏感性分析纳入附录

### 方向 C：回复级新中介变量

**价值**：很高  
**优先级**：P1

建议变量：

1. 步骤数
2. 代码块数
3. 引用/链接数
4. 表格数
5. markdown 结构深度

原因：

- 当前 SEM 已说明“长度 + 标题密度 + 粗体密度”并列影响偏好
- 但这些变量仍较粗粒度，尚不足以解释“结构化质量”本身

若加入上述变量，项目可以从“长度/格式偏好”进一步升级为“结构化表达偏好”的机制研究。

### 方向 D：`no_category` 的再细分

**价值**：中高  
**优先级**：P2

原因：

- `no_category` 体量最大，但语义异质性也最大
- 它很可能混合了自然语言解释、闲聊、伪代码和其他未标注任务

建议实施：

1. 先用 `code_bool` 拆成 `no_category_code` / `no_category_natural`
2. 再基于 embedding 或关键词聚类做弱监督细分

### 方向 E：处理效应的连续剂量建模

**价值**：中等  
**优先级**：P2

原因：

- 当前 C19/C20 采用“是否更长”“是否更高格式密度”的二值处理定义
- 这有利于解释，但会压缩剂量信息

建议实施：

1. 对 token_diff_ab 使用 spline 或 GAM
2. 对 header/bold/list density 使用 dose-response 曲线
3. 检查是否存在明显阈值效应或边际收益递减

### 方向 F：模型家族异质性分析

**价值**：中等  
**优先级**：P2

原因：

- 不同模型家族的输出习惯不同
- “长度偏好”可能部分是“特定家族输出风格”的交互结果

建议实施：

1. 给模型增加 family 标签
2. 做 family-level interaction
3. 检查 GPT / Claude / Gemini / Qwen / DeepSeek 等家族在偏好上的异质性
